# hackathon_team_a

## チーム開発の開発方針
### Unityのシーン管理について
 * シーンごとに担当の人を割り当てて競合を防ぐ
 * 他の人の担当のシーンを編集するときはその人と協力すること
 * 自分のシーンに別のシーンを呼び出すのはOK
 * 新しいシーンを作成するのはOK
 * 新しいシーンをビルド設定に追加するときは全員でブランチをマージする

### ブランチ管理について
 * 基本作業するときは自分のブランチで行う
 * mainブランチで作業してはいけない
 * 高頻度で定期的にコミット＆プッシュすること
 * 作った機能が完成したときにmainブランチにプルリクを飛ばす
 * mainブランチが更新されたときに自身のブランチにマージすること
